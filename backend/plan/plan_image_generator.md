### 🔥 추천 모델 (품질 A)
1. `stabilityai/stable-diffusion-xl-base-1.0` + `stabilityai/stable-diffusion-xl-refiner-1.0`
   - SDXL 표준 조합. 품질 안정적이고 범용 텍스트 → 이미지에 가장 무난

3. `segmind/SSD-1B`
   - SDXL 계열 경량 모델. 속도/자원 효율은 좋지만 품질은 1번보다 낮음

* **작업 내용:** `case1_sdxl.py` 작성.
* **도입 이유:** DALL·E 3 권한 이슈 대응.
* **발생한 문제:** -
* **해결 방법:** -

### 📍 3단계: 결과물 제어 및 파일 저장
* **작업 내용:** 결과물을 `backend/assets/generated`에 저장하고 `path` + `url(/static/generated/...)` 반환.

### 📍 4단계: Case1 단독 테스트

* **발생한 문제:** CUDA 드라이버 버전 불일치, 77 토큰 경고.
* **해결 방법:** torch CUDA 버전 재설치(cu121 등), 프롬프트 간소화.

### 📍 5단계: Case2 baseline (style_transfer)
* **작업 내용:** `case2_sdxl.py`로 프리셋 + 프롬프트 테스트.
* **성공:** 톤/조명 일부 반영.
* **실패:** 분석 텍스트 영향 미약, 구조 왜곡 발생.

### 📍 6단계: Case2 IP-Adapter
* **작업 내용:** `case2_ip_adapter.py` 도입.
* **성공:** 질감/조명 퀄리티 개선.
* **실패:** 광고 문구 삽입 실패, 구도 유지 불완전.

### 📍 7단계: Case3 설계 (예정)
* **목표:** 사용자 제품 이미지를 유지하고 배경만 자연스럽게 교체.
* **구현 방향:** `rembg`로 배경 마스크 생성 → SDXL Inpainting으로 배경 교체.
* **시행착오:** ControlNet(depth)는 구조는 유지하나 질감이 변형되는 문제가 있어 Inpainting으로 전환.

## ✅ 테스트 로그 (누적, 상세)

- **Test1 프롬프트 (Case1)**  
  입력: 기본 더미 프롬프트  
  성공: 이미지 생성/저장 정상  
  실패/이슈: CLIP 77 토큰 경고(일부 잘림)

- **Test2 SDXL (Case2 baseline)**  
  입력: 프리셋 이미지 + 분석 텍스트 + 무드  
  성공: 조명/스타일 반영  
  실패/이슈: 구조 왜곡, 분석 텍스트 영향 약함

- **Test3 한글 프롬프트 (Case2)**  
  입력: 한글 프롬프트 + 무드  
  성공: 색감/톤 유사  
  실패/이슈: 문구 삽입 실패, 형태 왜곡

- **Test4 IP-Adapter (Case2)**  
  입력: 프리셋 이미지 + 한글 프롬프트  
  성공: 질감/조명 품질 개선  
  실패/이슈: 문구 삽입 실패, 구도 유지 불완전

- **Test5 ControlNet(depth) (Case3)**  
  입력: 제품 이미지 + 배경 프롬프트  
  성공: 형태 유지  
  실패/이슈: 질감/재질이 페인팅 느낌으로 변형

- **Test6 ControlNet Img2Img (Case3)**  
  입력: 제품 이미지 + 배경 프롬프트  
  성공: 질감 유지 개선  
  실패/이슈: 배경 변화가 약함

- **Test7 Inpainting (Case3)**  
  입력: 제품 이미지 + 배경 교체 프롬프트  
  성공: 배경 교체 가능  
  실패/이슈: 프롬프트에 “배경 교체”를 명시해야 효과적

## 🧪 사용 모델/기술 스택 요약 (시도 내역)

| 기술/모델 | 역할 | 특징/결과 |
|---|---|---|
| SDXL Base+Refiner | 텍스트→이미지 | 고품질 생성, Case1 핵심 |
| SDXL Img2Img | 레퍼런스 스타일 | 구조 약함, 배경/질감 변형 발생 |
| IP-Adapter | 스타일 반영 | 질감/톤 개선, 문구 삽입 실패 |
| ControlNet(depth) | 형태 보존 | 구조 유지 강함, 질감 변형 큼 |
| ControlNet + Img2Img | 구조+질감 | 질감 유지↑, 배경 변화↓ |
| SDXL Inpainting | 배경 교체 | 배경 교체 안정적, 마스크 품질/프롬프트 의존 |
| rembg | 배경 분리 | 제품/배경 분리용 마스크 생성 |

## 🧩 Case4 계획: 사용자 이미지 기반 배경 변경 / 합성 보정

**목표:**  
사용자가 직접 올린 이미지를 기준으로 **배경만 바꾸거나** 합성 보정하는 기능.

**기본 전략 (단순·실전형, 부가코드 최소화):**  
- `SDXL Inpainting` 기반  
- 전제: **사용자가 마스크를 함께 제공**(프론트에서 영역 선택)  
- 마스크 없는 자동 분리는 **차후 단계**로 분리

**진행 단계(안):**
1. 입력 스펙 정의: `user_image`, `mask_image`, `prompt`, `format_type`
2. Inpainting 파이프라인 구성:  
   `diffusers/stable-diffusion-xl-1.0-inpainting-0.1`
3. 생성 결과 저장 및 반환 규격 통일  
4. 단독 테스트 (더미 이미지 + 간단 마스크)

**기대효과:**  
배경 변경/합성 작업을 최소 코드로 안정적으로 수행 가능
