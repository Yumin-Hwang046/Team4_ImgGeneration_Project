# 📖 시각 분석기 (Image Analyzer) 개발 노트

이 문서는 파이프라인 개발 과정 중 특히 "왜 이런 기술과 코드를 사용했는지"를 AI 입문자의 시선에서 정리한 기록입니다. 본 내용은 시각 분석기와 더불어 파이프라인 전반의 기본 지식을 담고 있습니다.

---

## 1. 라이브러리(도구)는 왜 이것들을 설치했는가?

| 라이브러리 | 설치 이유 |
|:---:|---|
| **`openai`** | 파이썬에서 OpenAI 서버(GPT)와 대화하려면 복잡한 인터넷 통신 규칙을 지켜야 합니다. 이 라이브러리는 그 복잡한 과정을 `client.chat.completions.create()` 같은 아주 쉬운 명령어 한 줄로 퉁치게 해주는 공식 리모컨입니다. |
| **`python-dotenv`** | API 키는 남에게 유출되면 절대 안 되는 소중한 비밀번호입니다. 코드 파일 한가운데에 비밀번호를 적어두면 해킹이나 실수로 사이트에 유출될 위험이 큽니다. 따라서 `.env`라는 별도의 비밀 금고 파일에 키를 숨겨두는데, 이 금고를 안전하게 열어 코드에 전달해 주는 열쇠 역할을 합니다. |
| **`Pillow`** | 차후에 이미지가 너무 크거나 용량이 거대할 경우, OpenAI 서버에 전송하기 직전에 이미지 크기를 리사이징/압축하는 등 "파이썬에서 이미지 자체를 조작"할 일이 생깁니다. 이때 쓰기 위해 전 세계적으로 가장 많이 사용하는 표준 이미지 처리 도구인 Pillow를 미리 깔았습니다. |

## 2. 사진 변환기(`encode_image` 함수)는 왜 필요한가?

우리가 카카오톡으로 친구에게 사진을 보낼 때는 사진 파일(JPG)을 그대로 전송합니다. 하지만 AI(컴퓨터) 특히 `GPT-4o Vision API`는 사진 파일을 통째로 받지 못하고 글자(텍스트 데이터)로 된 구조화된 박스에 담아달라고 요구합니다.

컴퓨터가 인터넷 선을 통해 이미지를 전송하기 가장 좋은 방법은, 이미지를 **길고 복잡한 글자(텍스트)**로 번역해서 넘겨주는 것입니다. 이 방식을 **Base64 인코딩**이라고 부릅니다. 

**코드 동작 원리:**
1. `open(image_path, "rb")`: 파이썬이 로컬 드라이버의 사진을 열어서 0과 1의 집합(바이트)으로 낱낱이 해체합니다. (`rb`는 읽기 전용 바이트라는 뜻)
2. `base64.b64encode(...)`: 해체된 0과 1의 덩어리들을 영어 알파벳 대소문자와 숫자로 이루어진 아주 긴 문장 형식으로 암호화(인코딩)합니다.
3. `.decode('utf-8')`: 마지막으로 파이썬 문자열 객체로 변환하여 에러 없이 반환합니다.

결과적으로 사진 파일이 `iVBORw0KGgoAAAANSUhEUgAA...` 와 같은 긴 문자열로 번역되며, 이 문자열 덩어리를 OpenAI 서버에 던져주어야 AI가 "아, 이 글자들을 다시 조합하면 이미지가 되는군!" 하고 이미지를 눈으로 '보게' 됩니다.
